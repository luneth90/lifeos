# 批改与更新协议

> 从 `SKILL.zh.md` 提取的完整批改流程与更新规则。

## 阶段2.5：批改流程

用户完成作答后触发（用户说"批改"、"改卷"、"检查复习"、"改一下"等）：

1. 读取复习文件中的用户作答
2. 逐题评估：
   - ✅ **正确**：简短确认 + 可补充延伸点（1-2 句）
   - ⚠️ **部分正确**：指出正确部分 + 补充遗漏关键点
   - ❌ **错误/遗忘**：给出正确解析 + 说明为何重要
3. 将批改结果写入复习文件的 `## 批改结果` 区块：

```markdown
## 批改结果

**成绩：** X/N（XX%）
**结果：** pass / fail

---

**Q1 ✅：** [简短确认 + 延伸]

**Q2 ⚠️：** [正确部分 + 遗漏补充]

**Q3 ❌：** [正确解析 + 重要性说明]

---

**掌握情况：**
- ✅ 已掌握：[概念列表]
- ⚠️ 部分掌握：[概念列表]
- ❌ 需加强：[概念列表]
```

4. 更新复习文件 frontmatter：`status: graded`，填入 `score` 和 `result`
5. 进入阶段3

---

## 阶段3：更新与总结

### 更新笔记 status

根据批改前的知识笔记状态更新 `{知识目录}/` 对应笔记；分数只用于记录弱项和判断后续掌握门槛：

| 批改前状态 | 条件 | 批改后状态 |
| --- | --- | --- |
| `review` | 首次复习已完成全部批改 | `revised`（与分数无关） |
| `revised` | 用户明确发起后续独立复核，成绩 ≥80%，且此前弱项全部通过 | `mastered` |
| `revised` | 未满足上述全部条件 | 维持 `revised` |
| `mastered` | 用户明确重测 | 维持 `mastered` |

> **规则**：知识状态只升不降（`draft → review → revised → mastered`）。`draft` 不进入 `/revise`；首次完整批改不得从 `review` 直接跳到 `mastered`。

### 更新项目文件掌握度小圆点

批改完成后，找到对应的 `{项目目录}/` 项目文件，更新内容规划中掌握度总览表格里对应章节的小圆点：

```
⚪ 未学    → 笔记不存在
🔴 整理中  → status: draft
🟠 待复习  → status: review
🟡 已复习待巩固 → status: revised
🟢 已掌握  → status: mastered
```

### 写入今日日记

在 `{日记目录}/YYYY-MM-DD.md` 的日志区追加（若文件存在）：

```markdown
- 复习 [[NoteTitle]]：[X]/[N] 题正确，[弱点概念] 需继续加强
```

### 输出复习摘要

```markdown
## 复习批改完成 📚

**复习范围:** [[NoteTitle]]
**模式:** 提问模式 | 费曼模式 | 盲点扫描
**成绩:** X/N 题正确（XX%）

**掌握情况:**
- ✅ 已掌握：[概念列表]
- ⚠️ 部分掌握：[概念列表]
- ❌ 需加强：[概念列表]

**笔记状态:**
- [[NoteTitle]] → revised / mastered（按本协议更新）

**项目进度:**
- [[项目名]] 掌握度表格已更新

**建议:**
- [下次复习重点，针对 ❌ 和 ⚠️ 概念]
- [若有疑问深化需求：可用 /brainstorm 或 /ask 探索]
```
