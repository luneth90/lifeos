# 技能完成报告格式规范

所有技能在执行完成后输出的报告应遵循统一结构，便于用户快速扫描结果。

## 标准结构

```markdown
## [动作] 完成

**已创建/修改:**
- [[文件名]] — 路径: `<相对路径>`
- [[文件名]] — 路径: `<相对路径>`

**状态更新:**
- [[来源文件]] → status 已更新为 `<新状态>`
- （若无状态变更则省略此节）

**建议下一步:**
- [关联技能建议，如 `/review` 复习、`/archive` 归档]
- （仅在有明确后续动作时列出）
```

## 格式规则

- 标题使用 `## [动作] 完成`，不加 emoji
- 文件引用使用 wikilinks `[[文件名]]` + 路径
- 状态变更明确标注来源和目标状态
- 建议下一步仅列出最相关的 1-3 个技能
- 不重复完整文件内容（除非用户要求）
- 使用中文输出
