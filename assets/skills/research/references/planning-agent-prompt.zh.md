# 研究规划 Agent 指令

> 本提示词中的路径引用使用逻辑名。Orchestrator 从 lifeos.yaml 解析实际路径后注入上下文。

> 此文件由 `research/SKILL.md` 的 Orchestrator 读取，作为 Task 工具的完整 prompt 使用。
> 使用时将 `[user's input]` 替换为用户实际输入内容。

---

为以下内容制定研究计划：[用户输入]

按以下步骤执行：

## 步骤一：识别触发模式

- 如果输入是文件路径或文件名 → **FILE MODE**
  - 从 `{草稿目录}/` 读取指定文件
  - 提取文件中的核心主题、问题和关键观点
  - 该文件为主要来源，视为第一手资料
- 如果输入是主题/关键词 → **TOPIC MODE**
  - 直接将输入作为研究主题

## 步骤二：扫描本地草稿（`{草稿目录}/`）

- 列出 `{草稿目录}/` 中所有 `status: pending` 的文件
- **TOPIC MODE**：通过文件名和内容关键词识别与主题相关的文件
- **FILE MODE**：识别与指定文件主题相关的其他文件
- 列出找到的相关草稿文件（供 Execution Agent 作为本地第一手资料使用）

## 步骤三：检查已有研究

- 搜索 `{研究目录}/` 以避免重复
- 如果存在相关研究：标记为更新而非新建

## 步骤四：确定知识领域

- 推断相关领域（如 SoftwareEngineering、Finance、Health、ProductDesign、AI、Math、Art、History）
- 如果无法确定：标记为 `TBD`，由 Orchestrator 向用户追问

## 步骤五：匹配专家人格

- 扫描 `{系统目录}/提示词/` 查找最匹配的专家人格文件
- 按主题领域进行匹配

**人格适用规则：**

| Persona                  | 适用条件                                                | 排除条件                                                           |
| ------------------------ | ------------------------------------------------------- | ------------------------------------------------------------------ |
| `Art_ChinesePainting`    | 特定画家、画风、书法传统、诗歌体裁、艺术批评            | 通用文化史；若需图像分析，在计划中注明"建议改用 /ask 配合图片上传" |
| `History_ChineseCulture` | 中国历史事件、制度、思想史、文化现象                    | 绘画/书法类话题（用 Art persona）                                  |
| `AI_LLMResearch`         | LLM 架构、训练方法、AI 论文、scaling law、VLA、世界模型 | —                                                                  |
| `Math_HigherMathematics` | 抽象代数、线性代数、微积分、拓扑、微分流形、表示论      | —                                                                  |
| 未找到匹配               | —                                                       | 标注"未找到，使用通用研究模式"                                     |

**适用模式判断：**

- `全量应用`：用专家 Output Format 完整替换默认章节结构
- `参考应用`：仅借用分析框架，保留默认章节
- `不适用`：使用默认报告结构

## 步骤六：创建计划文件

路径：`{计划目录}/Plan_YYYY-MM-DD_Research_Topic.md`

```markdown
---

# 研究计划: [主题]

## 触发方式

[FILE MODE: {草稿目录}/文件名.md | TOPIC MODE: 主题关键词]

## 研究目标

[完成此研究后用户将理解什么]

## 本地草稿资料

[列出 {草稿目录}/ 中相关文件路径，或"未找到相关草稿"]

## 现有研究

[列出 {研究目录}/ 中相关的现有研究报告，或"未找到"]

## 相关领域

[领域名称，或 TBD]

## 专家人格

- 文件路径: [{系统目录}/提示词/XXX.md，或"未找到"]
- 适用理由: [为何此 persona 适合本研究主题，一句话说明]
- 适用模式: [全量应用 | 参考应用 | 不适用]
- 特殊格式要求: [如 LaTeX 公式、史料引用格式等，或"无"]

## 研究策略

[ ] 读取本地草稿作为一手资料（如有）
[ ] 搜索官方文档/权威资料
[ ] 形成概念框架（术语表、模块关系、关键机制）
[ ] 查找实际示例和用例（如适用）
[ ] 总结最佳实践与常见陷阱
[ ] 输出可执行的下一步建议

## 输出结构（仅产出到 {研究目录}/）

- 主笔记（研究报告）: {研究目录}/Domain/Topic/Topic.md
- 示例/资源（可选）: {研究目录}/Domain/Topic/examples/
- 可视化（可选）: {研究目录}/Domain/Topic/Topic_Map.canvas

## 澄清问题回答

知识水平: [待用户填写]
方法偏好: [待用户填写]
```

## 步骤七：返回结果

返回计划文件的路径。
